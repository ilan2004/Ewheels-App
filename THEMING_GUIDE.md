# BrandColors Theming Guide

## Overview
This document outlines the BrandColors theming system for the EV Wheels App and provides guidance for applying consistent colors throughout the application.

## BrandColors Palette

```typescript
export const BrandColors = {
  surface: '#f4f3ef',   // navbar / light surfaces
  shellDark: '#2c2d32', // bottom tab bar / dark chrome
  primary: '#ff795b',   // main accent / active icon
  title: '#387868',     // title / brand text
  ink: '#1e1d19',       // dark text/icons on light backgrounds
};
```

## Usage Guidelines

### Import Statement
```typescript
import { BrandColors } from '@/constants/design-system';
```

### Common Replacements

| Old Color | New BrandColors | Usage |
|-----------|-----------------|-------|
| `Colors.primary[600]` | `BrandColors.primary` | Buttons, links, accents |
| `Colors.neutral[900]` | `BrandColors.ink` | Primary text, dark text |
| `Colors.neutral[700]` | `BrandColors.title` | Headings, important text |
| `Colors.white` | `BrandColors.surface` | Backgrounds, light surfaces |
| `Colors.neutral[800]` | `BrandColors.shellDark` | Dark backgrounds, tab bars |

## Best Practices

1. **Always import BrandColors** when adding colors to components
2. **Use semantic naming** - prefer `BrandColors.primary` over hardcoded hex values
3. **Test on both light and dark themes** if your app supports theme switching
4. **Use opacity modifiers** for subtle variations: `BrandColors.ink + '80'` (50% opacity)
5. **Maintain consistency** across similar UI elements

## Migration Checklist

When updating a component:

- [ ] Import BrandColors from design-system
- [ ] Replace hardcoded hex colors with BrandColors
- [ ] Replace Colors.primary[xxx] with BrandColors.primary
- [ ] Replace Colors.neutral[xxx] with appropriate BrandColors
- [ ] Test the component visually
- [ ] Update any related TypeScript types if needed

## Colors to Avoid

- Hardcoded hex values like `#3B82F6`, `#1D4ED8`, etc.
- Generic `Colors.primary[600]` - use `BrandColors.primary` instead
- `Colors.neutral[900]` for text - use `BrandColors.ink` instead
